Salah satu bidang usaha yang sangat membutuhkan efisiensi manajemen data adalah bisnis Studio Photo. Layanan studio foto sering kali melibatkan transaksi yang kompleks, mulai dari pengelolaan berbagai paket foto (terdiri dari nama paket, ukuran cetak, dan harga), manajemen diskon promo, hingga pencatatan pembayaran dengan metode bayar non-tunai (transfer bank, QRIS) maupun tunai.Selama ini, pencatatan transaksi di studio foto sering kali dilakukan secara manual atau menggunakan program spreadsheet sederhana yang rawan terhadap kehilangan data, kesalahan perhitungan biaya, serta menyulitkan proses rekapitulasi omzet berkala. Terlebih lagi, akses terhadap data laporan keuangan tidak memiliki batasan wewenang yang jelas antara kasir yang menginput data dan manajer yang memantau profitabilitas toko.Untuk mengatasi permasalahan tersebut, dirancanglah sebuah aplikasi desktop terintegrasi bertajuk "PhotoStudio". Aplikasi ini dibangun menggunakan bahasa pemrograman Java dengan arsitektur Model-View-Controller (MVC), Data Access Control ( DAO ) serta penyimpanan database relasional MySQL. Penerapan aplikasi ini diharapkan dapat mempercepat proses pelayanan kasir, menjaga akurasi kalkulasi biaya pemesanan foto, mempermudah manajemen data master bagi admin, serta menyajikan laporan keuangan harian yang valid bagi pengelola toko.1.2 Rumusan MasalahBerdasarkan latar belakang di atas, rumusan masalah yang diajukan dalam proyek ini adalah:Bagaimana merancang dan membangun Aplikasi Studio Photo berbasis desktop menggunakan bahasa pemrograman Java dengan basis data MySQL?Bagaimana mengimplementasikan arsitektur Model-View-Controller (MVC) dan Data Access Object (DAO) untuk mempermudah operasi CRUD (Create, Read, Update, Delete) pada sistem?Bagaimana merancang hak akses login multi-user dan menyajikan laporan ringkasan transaksi omzet harian berdasarkan hak akses tersebut?1.3 TujuanTujuan dari pengerjaan tugas akhir ini adalah:Menghasilkan Aplikasi Studio Photo berbasis desktop yang dapat mempermudah operasional pelayanan foto dan manajemen transaksi kasir.Menerapkan konsep pemrograman berorientasi objek (OOP) serta pola MVC dan DAO untuk membuat struktur kode Java yang modular, rapi, dan mudah dipelihara.Menyediakan antarmuka laporan interaktif bagi pihak manajemen untuk memantau omzet harian dan detail pesanan secara real-time.1.4 ManfaatManfaat yang didapatkan dari pembangunan aplikasi ini adalah:Bagi Pengguna (Studio Photo):Meningkatkan Pemrograman Java merupakan bahasa berorientasi objek murni yang bersifat platform-independent (Write Once, Run Anywhere) melalui JVM (Java Virtual Machine). Java SE menyediakan kerangka kerja Swing untuk pengembangan Graphical User Interface (GUI) desktop yang kaya dengan komponen visual terstandarisasi seperti JFrame, JPanel, JButton, dan JTable. IDE NetBeans digunakan sebagai lingkungan pengembangan karena menyediakan visual GUI builder drag-and-drop yang sangat membantu dalam merancang tata letak antarmuka aplikasi secara presisi.2.2 Konsep yang DiterapkanBeberapa konsep pemrograman terstruktur dan modern yang diimplementasikan dalam pengembangan aplikasi PhotoStudio ini meliputi:Pemrograman Berorientasi Objek (Object-Oriented Programming - OOP)Class & Object: Kelas didefinisikan sebagai cetak biru model (misalnya model User, Order, PaketPhoto) yang kemudian diinstansiasi menjadi objek dinamis selama runtime program.Encapsulation (Pengkapsulan): Menyembunyikan detail representasi objek menggunakan akses kontrol 'private' pada variabel anggota, dan membukanya secara aman melalui metode getter dan setter.Inheritance (Pewarisan): Digunakan secara implisit pada Swing GUI di mana setiap frame kustom mewarisi perilaku dari kelas induk javax.swing.JFrame.Model-View-Controller (MVC) & Data Access Object (DAO) PatternPola arsitektur MVC memisahkan logika aplikasi menjadi tiga komponen utama:Model: Representasi data bisnis studio foto (misalnya kelas Order.java).View: Antarmuka visual yang berinteraksi langsung dengan pengguna (misalnya Kasir.java).Controller: Jembatan yang menangani interaksi pengguna dari View dan memperbarui Model (misalnya KasirController.java).Sedangkan DAO (Data Access Object) menyembunyikan detail operasional query database relasional dari controller. Data access seperti INSERT, SELECT, dan UPDATE dipusatkan pada kelas DAO (misalnya OrderDAO.java).Operasi CRUD (Create, Read, Update, Delete)CRUD merupakan empat pilar utama dalam pemanipulasian data persisten di database. Pengguna dengan hak akses Admin dapat memasukkan data baru (Create), menampilkan daftar data di tabel (Read), mengubah data yang sudah ada (Update), serta menghapus data master (Delete) menggunakan dialog GUI yang interaktif.2.3 Penyimpanan DataAplikasi ini memanfaatkan database relasional MySQL sebagai media penyimpanan data persisten yang tangguh. Koneksi dari aplikasi Java ke MySQL dijembatani oleh JDBC (Java Database Connectivity) API. Kelas DatabaseConnection diimplementasikan menggunakan desain singleton-like di mana koneksi dipusatkan pada satu instance Connection untuk menghemat pemakaian sumber daya database. Komunikasi query SQL dilakukan secara aman menggunakan PreparedStatement untuk mencegah celah keamanan SQL Injection.BAB IIIANALISIS & PERANCANGAN3.1 Analisis KebutuhanAnalisis kebutuhan dilakukan untuk memetakan fitur apa saja yang mutlak harus dimiliki oleh aplikasi (fungsional) dan kriteria kualitas sistem pendukungnya (non-fungsional):Kebutuhan Fungsional:Sistem Login: Menyaring hak akses berdasarkan 3 role berbeda (Admin, Kasir, Kepala Toko).Manajemen Data Master (Admin): CRUD User, CRUD Paket Foto, CRUD Diskon, dan CRUD Metode Bayar.Modul Transaksi Kasir: Pembuatan order transaksi baru, kalkulasi otomatis subtotal, diskon persentase, total bayar, kembalian uang, serta generator nomor transaksi otomatis (format ORD-YYYYMMDD-XXX).Modul Laporan Manager: Rekapitulasi omzet total, jumlah pesanan, jumlah pelanggan harian, serta pencarian ringkasan omzet berdasarkan rentang tanggal tertentu.Kebutuhan Non-Fungsional:Usability (Kemudahan Penggunaan): Aplikasi dirancang dengan warna antarmuka yang bersih (Nimbus Look and Feel) dan tata letak menu yang intuitif.Security (Keamanan): Login berpassword untuk membatasi fungsionalitas berdasarkan wewenang.Data Integrity (Integritas Data): Data transaksi dikaitkan dengan integritas relasi tabel di database MySQL.3.2 Rancangan DataRancangan tabel database db_studiofoto_231011401688 dijabarkan secara rinci sebagai berikut:1.
